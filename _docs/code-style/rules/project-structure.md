## Document and Draft Placement Guidelines

- All non-runtime and non-deployment documents, including requirements, design notes, TODOs, notes, research, drafts, and runbooks, must be placed under the workspace-level `_docs/` directory.
- Each project root should only contain files required for runtime, deployment, and agent configuration. Project-specific rules are defined by that repo's `AGENTS.md`.
- Project API documentation must be placed at `_docs/_API_vX.X.md`.
